# Frontend Directory Structure

Use `web/default` for new frontend work unless the task explicitly names
`web/classic`.

## Primary Frontend: `web/default`

| Path | Responsibility |
| --- | --- |
| `src/main.tsx` | React root, QueryClient defaults, router creation, global providers, initial branding |
| `src/routes/` | TanStack Router file routes, layout routes, auth/setup guards, public pages |
| `src/features/<feature>/` | Feature-owned API functions, types, constants, hooks, components, dialogs, tables, entry component |
| `src/components/ui/` | Reusable Base UI/shadcn-style primitives and wrappers |
| `src/components/` | Cross-feature components such as data tables, layout, command menu, loading/error states |
| `src/components/layout/` | App/public layout components, workspace navigation, sidebars, page section layout |
| `src/stores/` | Zustand stores, localStorage-backed auth/system/notification state |
| `src/lib/` | Shared API client, utilities, constants, date setup, DOM helpers, error handling |
| `src/hooks/` | Shared React hooks |
| `src/context/` | Theme, font, direction, layout, and search providers |
| `src/i18n/` | i18next config, locale JSON files, static translation keys |
| `src/styles/` | Global styles and Tailwind/CSS variable setup |
| `rsbuild.config.ts` | Rsbuild entry, aliases, dev proxy, TanStack Router plugin, chunk splitting |

Feature folders usually expose:

- `api.ts` for HTTP calls
- `types.ts` for request/response/UI types
- `constants.ts` for local enums, option metadata, and translation keys
- `index.tsx` for the feature screen
- `components/`, `hooks/`, and `lib/` only when needed

Reference files: `web/default/src/features/channels/api.ts`,
`web/default/src/features/channels/types.ts`,
`web/default/src/features/models/index.tsx`,
`web/default/src/features/models/section-registry.tsx`.

## Routes

Routes live in `web/default/src/routes` and use TanStack Router file routing:

- Public pages are direct route files such as `index.tsx`, `pricing/index.tsx`,
  `about/index.tsx`, and legal pages.
- Auth pages live under `(auth)`.
- Error pages live under `(errors)`.
- Authenticated app pages live under `_authenticated`.
- Nested sections use `$section` route params, for example
  `_authenticated/models/$section.tsx`.

Do not edit `routeTree.gen` manually. It is generated by the TanStack Router
plugin configured in `rsbuild.config.ts`.

## Legacy Frontend: `web/classic`

`web/classic` remains embedded by Go and uses React 18, Vite, Semi UI, plain JS
and JSX, and helper modules under `src/helpers`. When changing classic:

- Match the existing JS/JSX style.
- Use `web/classic/package.json` scripts.
- Keep classic-specific helper changes under `web/classic/src`, not
  `web/default/src`.

## Electron Shell

`electron/` contains a small shell/tray wrapper and package metadata. Do not mix
Electron concerns into the web frontends.
